# 🤖 Project Agents & Skills Directory

This document serves as a manifest of all specialized agents and skills utilized in this project. It helps new contributors understand what capabilities are available and how to invoke them for specific tasks.

## Overview
We use specialized agent skills to maintain high quality, adhere to best practices, and avoid generic AI output patterns. Each skill is designed for a particular domain (e.g., UI/UX design, Cloudflare architecture, Authentication).

## ⚙️ Available Core Skills

The following skills are available in the project's scope:

*   **`frontend-design`**: For general component creation and beautiful UIs.
*   **`gpt-taste`**: Advanced motion engineering and high-fidelity UI/UX.
*   **`high-end-visual-design`**: Enforces premium, polished visual standards (typography, spacing).
*   **`minimalist-ui`**: For clean, editorial, monochrome design systems.
*   **`industrial-brutalist-ui`**: For data-heavy, mechanical, terminal-style dashboards.
*   **`stitch-design-taste`**: Semantic Design System for Google Stitch, ensuring modern UI patterns.
*   **(Add other skills here as they are added to the project)**

## 🔐 Authentication & Identity
*   **`create-auth-skill`**: Scaffolds and implements full authentication flows (login/signup).
*   **`better-auth-best-practices`**: General guidance for betterauth setup.
*   **`organization-best-practices`**: For multi-tenant, team, and RBAC setup.
*   **`two-factor-authentication-best-practices`**: Implementing MFA/2FA.

## ☁️ Cloud Infrastructure (Cloudflare)
*   **`cloudflare`**: General purpose skill for Workers, Pages, KV, R2, AI, etc.
*   **`durable-objects`**: For building stateful services (chat rooms).
*   **`wrangler`**: Wrapper for the Cloudflare CLI and deployment best practices.

## 🚀 Workflow & Architecture
*   **`system-design`**: To architect overall systems or define APIs.
*   **`babysit`/`call`**: For orchestrating complex, multi-step workflows.
